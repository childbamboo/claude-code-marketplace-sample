# テストコード生成

既存のコンポーネントや関数に対するテストコードを自動生成します。

## 生成対象

テストを生成したいファイルやコンポーネントを指定してください：

- コンポーネントファイルのパス
- 関数やモジュールの名前
- ディレクトリ（配下の全ファイル）

## 生成されるテストの種類

### 1. ユニットテスト
- コンポーネントのレンダリングテスト
- Props の検証
- 状態変化のテスト
- イベントハンドラーのテスト
- 関数の入出力テスト

### 2. インタラクションテスト
- ユーザーイベント（クリック、入力など）
- フォームの送信
- バリデーションエラー
- 非同期処理の完了待機

### 3. スナップショットテスト
- 初期レンダリング
- 異なるPropsでのレンダリング
- 状態変化後のレンダリング

### 4. アクセシビリティテスト
- ARIA属性の確認
- キーボードナビゲーション
- スクリーンリーダー対応

## テストフレームワーク

対応するフレームワーク：
- **Jest** + **React Testing Library** (React)
- **Vitest** + **Testing Library** (Vite)
- **Vue Test Utils** (Vue)
- **Playwright** (E2E)
- **Cypress** (E2E)

## 生成内容

```typescript
  it('renders with correct text', () => {
    // レンダリングテスト
  });

  it('calls onClick when clicked', () => {
    // イベントハンドラーテスト
  });

  it('is disabled when disabled prop is true', () => {
    // Props検証
  });

```

## ベストプラクティス適用

- AAA パターン（Arrange, Act, Assert）
- 明確なテスト名
- 独立したテストケース
- 適切なモックとスタブ
- カバレッジの確保

テストを生成したい対象を教えてください。
