# セキュリティチェック

バックエンドコードのセキュリティ脆弱性をチェックし、改善提案を行います。

## チェック項目

### 1. 認証・認可
- [ ] パスワードの安全な保存（ハッシュ化）
- [ ] JWT/セッショントークンの適切な管理
- [ ] 権限チェックの実装
- [ ] パスワードポリシーの適用

### 2. インジェクション攻撃対策
- [ ] SQLインジェクション対策
- [ ] NoSQLインジェクション対策
- [ ] コマンドインジェクション対策
- [ ] XSS（クロスサイトスクリプティング）対策
- [ ] パラメータ化クエリの使用

### 3. データ保護
- [ ] 機密情報の暗号化
- [ ] HTTPS通信の強制
- [ ] 環境変数の適切な管理
- [ ] APIキー/シークレットの安全な保存
- [ ] 個人情報の適切な取り扱い

### 4. API セキュリティ
- [ ] レート制限の実装
- [ ] CORS設定の確認
- [ ] 入力バリデーション
- [ ] 出力エンコーディング
- [ ] HTTPヘッダーのセキュリティ設定

### 5. 依存関係の脆弱性
- [ ] 古いパッケージの確認
- [ ] 既知の脆弱性チェック
- [ ] セキュリティアップデートの適用

### 6. ログとモニタリング
- [ ] セキュリティイベントのログ記録
- [ ] 機密情報のログ除外
- [ ] 異常アクセスの検知

### 7. エラーハンドリング
- [ ] 詳細なエラー情報の非公開
- [ ] スタックトレースの非表示
- [ ] 適切なエラーメッセージ

## 実行内容

1. **コードスキャン**: セキュリティ脆弱性の検出
2. **ベストプラクティスチェック**: OWASP Top 10 に基づく確認
3. **改善提案**: 検出された問題の修正方法
4. **パッチ適用**: 可能な場合は自動修正

チェックしたいファイルやディレクトリを教えてください。
全体的なセキュリティレビューも実施できます。
