# API エンドポイント作成

新しいAPIエンドポイントの実装をサポートします。

## エンドポイント情報

以下の情報を教えてください：

1. **HTTPメソッド**: GET, POST, PUT, PATCH, DELETE
2. **パス**: 例）/api/users, /api/products/:id
3. **用途**: どのような機能を提供するか
4. **認証**: 認証が必要か、権限レベルは？

## リクエスト/レスポンス仕様

- **リクエストボディ**: 必要なパラメータとバリデーション
- **レスポンス形式**: 成功時・エラー時のレスポンス
- **ステータスコード**: 使用するHTTPステータスコード

## 自動生成される内容

1. **ハンドラー実装**
   - ルーティング設定
   - コントローラー/ハンドラー関数
   - ミドルウェア適用

2. **バリデーション**
   - 入力バリデーション
   - スキーマ定義（Zod, Joi, yupなど）

3. **エラーハンドリング**
   - 統一されたエラーレスポンス
   - エラーロギング

4. **ドキュメント**
   - OpenAPI/Swagger仕様
   - JSDoc/コメント

5. **テストコード**
   - ユニットテスト
   - 統合テスト

## ベストプラクティス適用

- RESTful設計
- セキュリティ対策（SQLインジェクション、XSS等）
- レート制限
- CORS設定
- 適切なHTTPステータスコード使用

作成したいAPIエンドポイントの詳細を教えてください！
