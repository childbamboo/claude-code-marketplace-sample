# 仕様書作成コマンド

$ARGUMENTS の機能仕様書を作成してください。

## 仕様書に含めるべき項目

1. **概要**: 機能の目的と背景
2. **要件**: 機能要件と非機能要件
3. **仕様詳細**:
   - 入力/出力
   - データ構造
   - API仕様（該当する場合）
4. **制約事項**: 技術的制約やビジネス上の制約
5. **テストケース**: 主要なテストシナリオ
6. **実装方針**: 推奨される実装アプローチ

## 出力形式

Markdown形式で仕様書を作成し、適切なファイル名で保存してください。
