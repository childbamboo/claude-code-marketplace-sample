# 仕様書レビューコマンド

@$ARGUMENTS の仕様書をレビューしてください。

## レビュー観点

1. **完全性**: 必要な情報がすべて含まれているか
2. **明確性**: 仕様が明確で曖昧さがないか
3. **実装可能性**: 技術的に実装可能な仕様か
4. **一貫性**: 他の仕様や既存システムとの整合性
5. **テスト容易性**: テスト可能な仕様になっているか

## 出力形式

以下の形式でフィードバックを提供してください：

- **Critical（必須修正）**: 仕様の欠陥や重大な問題
- **Warning（修正推奨）**: 改善が望ましい点
- **Suggestion（提案）**: より良い仕様にするための提案
